# Phase 6: 宫位诊断函数 (House Diagnosis Function)

> 推理强度：🟡中（循环执行）
> 输入：Phase 1-5 全部卡片
> 输���：宫位诊断卡（每次一个宫位，可循环）
> 下一步：用户选下一个宫位，或进入 Phase 7

---

**触发方式:** 用户指定目标宫位（如 10宫-事业）后执行。可循环执行多个宫位。

## FUNCTION: House_Diagnosis_Function (Target_House)

### 1. 资产管理者审计 (The Manager Audit)

- **定位:** 目标宫位的宫头星 (House Lord / Bhavesha)
- **角色:** 该宫位资源的法定代表人 (Carrier)
- **审计项:**
  - 去向 (P5/P6): 它带着资源去了哪个宫位？路况如何？→ 变现路径
  - 身份 (P1): 忠诚者/交易者/掠夺者？→ 底层原始偏置
  - 动力 (P7/P8/P9): 车型、司机年龄、基础功率 → 主动掌控 or 宿命运行

### 2. 仓库租客审计 (The Tenant Audit)

- **定位:** 目标宫位内的驻扎行星 (Resident Planets)
- **角色:** 外部干扰 (Bias) 或附加负载 (Cargo)
- **审计项:**
  - 资源带入: 它是哪个宫位的宫主星？（如：11宫主入2宫 = 带着"收益"来租房）
  - 环境干扰: 吉星 = 增值，凶星 = 损耗或竞争压力
  - 相位干扰 (P10): 谁在隔空"空中加油"或"恶意拦截"

### 3. 硬件带宽审计 (The Hardware Audit)

- **定位:** 目标宫位的原始 SAV 分值
- **审计标准:**
  - SAV > 32: 顶级自动化仓库，躺赢模式
  - 26-32: 标准稳健仓库，多劳多得
  - 20-25: 高阻力仓库，入不敷出
  - < 20: 结构性漏雨，资产难以留存

### 4. 分盘扩展（按主题推荐）

根据用户指定的宫位/主题，自动推荐对应的分盘进行交叉确认：

| 主题 | 主盘宫位 | 推荐分盘 | 分盘用途 |
|------|---------|---------|---------|
| 婚姻/配偶 | 7宫 | **D9 Navamsha** | 配偶特质、婚姻质量 |
| 事业/职业 | 10宫 | **D10 Dashamsha** | 职业方向、社会地位 |
| 子女 | 5宫 | **D7 Saptamsha** | 子女数量、关系 |
| 父母/家世 | 4宫/9宫 | **D12 Dvadashamsha** | 父母状况、家族业力 |
| 教育/学业 | 4宫/5宫 | **D24 Chaturvimshamsha** | 学习领域、学术成就 |
| 财富 | 2宫/11宫 | **D2 Hora** | 财富积累能力 |
| 健康 | 1宫/6宫 | **D6 Shashthamsha** | 疾病模式 |
| 灵性/前世 | 12宫/5宫 | **D60 Shashtiamsha** | 前世业力（最微观） |

**分盘分析规则：** D1 是主判据，分盘只能确认/修正/否定 D1 的指示，不能独立给出 D1 没有的结论。
