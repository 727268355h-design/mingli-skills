## 目标

基于 Model A 确定的模式，按权重排序计算最终成果的规模。

## 算法选择（基于 Model A 模式判定）

| 模式 | 公式 | 核心逻辑 |
|------|------|----------|
| 创始人 | Scale ∝ P4×0.4 + P6×0.3 + P5×0.2 + P8×0.1 | 资源和路况是基本盘 |
| 吉祥物 | Scale ∝ P6×0.6 + P4×0.3 + P3.2×0.1 | 被环境推着走 |
| 雇佣兵 | Scale ∝ P6×0.3 + P4×0.3 + P5×0.2 + P1_bias×0.2 | 高效但 P1 偏置权重高（代价大）|
| 职业经理人 | Scale ∝ P6×0.5 + P9×0.3 + P7×0.2 | 平台决定流量 |
| 飘萍 | Scale ∝ P1×0.8 + P6×0.2 | 纯看命 |

*注：权重为设计选择非经验值。分类变量按以下映射转为 0-1 数值：*

### 分类变量数值映射

| 变量 | 值 → 数值 |
|------|----------|
| P1 身份 | Loyalist=1.0, Trader=0.5, Hostile=0.2 |
| P5 路段 | Kendra/Trikona=1.0, 2宫=0.9, Upachaya(3/11)=0.7, 6宫=0.5, 8宫=0.3, 12宫=0.2 |
| P7 尊贵度 | 入旺=1.0, MT=0.9, 入庙=0.8, 友=0.6, 中=0.5, 敌=0.3, 陷=0.1 |
| P8 司机 | Yuva=1.0, Kumara=0.8, Vriddha=0.4, Baala=0.3, Mrita=0.1 |
| P1_bias | Hostile 偏置强度 = (1 - P1_Hostile值) = 0.8（凶身份的系统误差权重）|

### 补丁
- 顶级 Yoga (如 Dharma-Karma Yoga)：Scale_Grade 提升 1-2 档
- Yoga 因"燃烧"或"战争"受损：增益减半或失效
