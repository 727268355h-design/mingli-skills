# Phase 7：综合报告 — 执行指令

> 前置：Phase 0-6 全部卡片已完成
> 输入：全部前序卡片
> 输出：最终解卦报告

> 推理强度：🟡中

## 7.1 交叉验证矩阵

逐项检查前序 Phase 之间的逻辑一致性：

| 检查项 | Phase A | Phase B | 一致？ |
|--------|---------|---------|--------|
| 用神取用与问事类型匹配 | Phase 1（问事类型） | Phase 2（用神） | ✅/❌ |
| 旺衰与月日一致 | Phase 0（月日干支） | Phase 3（旺衰表） | ✅/❌ |
| 生克分析引用旺衰 | Phase 3（旺衰） | Phase 4（生克） | ✅/❌ |
| 吉凶引用生克和旺衰 | Phase 3+4 | Phase 5（吉凶） | ✅/❌ |
| 应期方向与吉凶一致 | Phase 5（吉凶） | Phase 6（应期） | ✅/❌ |
| 特殊卦象是否影响了判断 | Phase 1（标记） | Phase 5（修正） | ✅/❌ |
| 伏神处理完整性 | Phase 2（伏神） | Phase 4（飞伏互作） | ✅/❌ |

**矛盾处理**：
- 如发现矛盾 → 回溯到矛盾源头Phase，重新检查
- 如无法解决 → 在报告中标注 ⚠️ 并说明矛盾点

## 7.2 禁止项检查

- [ ] 无极端断语（"必定大富大贵"/"一生悲惨"/"必死无疑"等）
- [ ] 无跳步（每个 Phase 都有产出）
- [ ] 无自相矛盾（前面说吉后面说凶无解释）
- [ ] 无骑墙（"一方面好一方面不好"没有优先级判断）
- [ ] 用神取用有查表依据（非凭感觉）
- [ ] 旺衰判断有查表依据（非凭感觉）

## 7.3 最终解卦报告

### 报告结构

```
━━━━━━━━━━ 六爻解卦报告 ━━━━━━━━━━

【卦象概述】
  主卦：[卦名] → 变卦：[卦名]
  占问：[问事内容]
  日期：[年月日干支]

【核心判断】
  用神：[六亲][地支]（[旺衰]）
  吉凶：[大吉/吉/小吉/平/小凶/凶/大凶] [确定性]
  应期：[干支] ≈ [公历]

【详细分析】

  1. 用神状态
     [用神的旺衰、动静、空亡月破等状态总结]

  2. 有利因素
     [原神能否生、日月是否生扶、有无贵人等]

  3. 不利因素
     [忌神能否克、有无空破墓绝、世爻状态等]

  4. 综合判断
     [综合以上分析，给出明确判断]
     [引用特殊卦象信号：六冲/六合/游魂/归魂]

  5. 时间推断
     [病药法推理过程]
     [应期结论]

  6. 建议
     [基于卦象给出具体建议]
     [注意事项和提醒]

【确定性标注】
  ✅ 确定：[列出高确信度结论]
  ⚠️ 不确定：[列出存疑之处]
  ❌ 矛盾：[列出矛盾信号及说明]

━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 回答问题原则

- **直接回答**：求测者问什么，先直接回答，再展开分析
- **不回避**：不用"有好有坏"这种骑墙话，给出明确方向
- **标注不确定**：真不确定就标 ⚠️，不伪装确定
- **给建议**：不只说结果，还说可以做什么
