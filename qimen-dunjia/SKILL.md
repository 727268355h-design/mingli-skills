# 奇门遁甲断卦 Skill

## 工作流

### 1. 获取输入

需要两个信息：
- **时间**：默认当前北京时间。用户可指定。格式 `YYYY MM DD HH`（24小时制）
- **问题**：用户想测什么事（"一物一太极"，聚焦一件事）

如果用户没给时间，用当前北京时间（澳洲 AEST -2 小时）：
```bash
python3 -c "from datetime import datetime, timedelta; t=datetime.utcnow()+timedelta(hours=8); print(t.strftime('%Y %m %d %H'))"
```

### 2. 排盘 + 自检（含失败处理）

排盘、自检、创建 trace 一步完成，144 项检查：

```bash
cd ~/Projects/qimen-engine && python3 -c "
import qimen, json, verify, trace

r = qimen.paipan(YEAR, MONTH, DAY, HOUR)
ok, errs, warns, checks = verify.verify_one(YEAR, MONTH, DAY, HOUR)

# 创建 trace 日志
tid = trace.create_trace('YEAR-MONTH-DAY HOUR:00', 'USER_QUESTION', r)
trace.log_verify(tid, ok, [str(e) for e in errs], checks)
print(f'trace_id: {tid}')

if not ok:
    print('❌ 排盘自检失败:')
    for e in errs: print(f'  {e}')
else:
    print(f'✅ 自检通过 ({checks}项)')
    qimen.print_pan(r)
    print()
    print(json.dumps(r, ensure_ascii=False, indent=2))
" 2>&1
```

**记住 trace_id**，后续每步都要用。

**自检失败 → 定位 → 修复 → 重验：**

1. **定位根因**：verify.py 的错误标签告诉你哪步出问题，按以下决策树修：

| 错误步骤 | 可能根因 | 修复动作 |
|----------|---------|---------|
| 四柱 | sxtwl 库版本/输入范围 | 检查年份是否在 sxtwl 支持范围(1900-2100) |
| 定局（节气） | 节气边界时刻判断 | 用 `getJieQiJD()` 打印精确交节时间，对比输入时间 |
| 定局（元/局数） | 拆补法符头计算 | 手动算日柱甲子序号 → 往前找甲/己日 → 查地支分类 |
| 地盘 | dipan_18ju.json 数据 | 查对应局的 JSON 条目是否完整 |
| 天盘偏移不一致 | rotate_tianpan 逻辑 | 打印 shi_gong、orig_gong、offset，看哪颗星偏移异常 |
| 人盘偏移不一致 | rotate_renpan 逻辑 | 打印 xun_gong、steps、ordered 路径，看哪步走偏 |
| 神盘顺序 | layout_shenpan 的阴阳遁方向 | 检查 dun 参数传入是否正确 |
| 伏吟反吟 | detect_all 计算逻辑 | 独立重算天盘干==地盘干，对比 |
| 值符落宫 | 甲遁+5宫处理 | 检查 shi_gan==甲时 shi_gong 是否做了5→2 |

2. **修复后重跑**：改完 qimen.py → `python3 verify.py YEAR MONTH DAY HOUR` 单盘验证 → 通过再继续
3. **如果是数据文件错误**（JSON表有误）：修 JSON → `python3 verify.py 200` 批量回归
4. **实在改不了的引擎 bug**：明确告诉用户"排盘引擎在此时间点有已知问题"，不输出不确定的结果

### 4. 九步断卦SOP

**逐步执行，每步从盘面JSON+知识库JSON提取数据，给出解读。**

**核心原则**：
- **判吉凶**：第零步~第六步 + 第五步**层1-5** + 第七步阶段A
- **断事象**：第五步**层6-7** + 第七步阶段B
- 先定吉凶，再充实事象。不可颠倒。

**R规则说明**：本SOP中引用的 R01-R21 规则，定义在 `duanjua_knowledge.json` 的 `断卦规则` 字段中。

#### 第二步：看盘外格局（R02一票否决检查）

从排盘JSON的 `四柱` 和 `检测.全局` 提取：

1. **一票否决检查**（R02，最高优先级）：
   - **硬否决**：五不遇时 / 六仪击刑 / 三奇入墓 → 遇之皆凶，无论其他因素多吉
   - **近否决**：天网四张 / 全局伏吟 / 全局反吟 → 极凶但有化解可能
   - 注意：单层伏吟/反吟仅为重大负面，不构成近否决

   **天网四张**：天盘庚加临地盘值符(甲/戊)所在之宫，庚金罩甲=天网覆盖。主谋事受阻、出行不顺。属近否决（非硬否决），有天显时格可化解。⚠️ 多流派对具体条件有分歧，以 `duanjua_knowledge.json` 中定义为准。

2. **五不遇时**：`检测.全局.五不遇时` → 若 True，触发硬否决
3. **三奇入墓检查**：查 `sanqi_xiangyi.json` → 乙墓在坤二宫(未)，丙墓在乾六宫(戌)，丁墓在艮八宫(丑)。天盘三奇落入对应墓宫=触发硬否决。⚠️ 乙奇入墓有流派分歧(部分说乾六宫/戌)，以 JSON 定义为准
4. **日干与时干关系**：`四柱.日干` vs `四柱.时干` 五行生克
5. **四柱地支关系**：年支/月支/日支/时支之间的冲合刑害（简要分析）

---

#### 第三步：看大环境（R05旺衰+R13大局+R11内外盘）

1. **月令旺衰**（R05，此步=记录状态，第七步=用它做终极裁判）：`检测.全局.旺衰` → 哪行旺/相/休/囚/死
   - 旺衰是所有吉凶判断的前提：落宫=能力（长期），月令=天时（临时）
   - 旺相+得令=最佳；衰弱+失令=最差
2. **伏吟反吟**（R13首重大局）：`检测.全局.星伏吟/星反吟/门伏吟/门反吟`
   - 伏吟 → 宜静不宜动，事慢，守为上，利主不利客
   - 反吟 → 事有反复，主变动，利客方
   - 在横看/主客场景下，伏吟反吟优先级最高
3. **内外盘**（R11）：`检测.全局.内盘/外盘` + 日干/时干落宫判断远近快慢
   - 内盘=近+快，外盘=远+慢（7源一致）
   - 内外盘与伏吟反吟是**两个独立维度**，不存在映射
   - 日时干一内一外时，按"远+慢"处理（只有两个都在内盘才算快）

---

#### 第四步：看值符值使（R08分场景优先级+R18值使裁决+R20二分法）

从 `值符值使` 提取：

1. **值符星**（`值符值使.值符星`）→ 大方向、大势、长远
2. **值使门**（`值符值使.值使门`）→ 具体执行、事情最终结果（R18裁决成败第一要素）
3. **核心用二分法**（R20）：值符=初始走向；值使=最终结局
4. **分场景优先级**（R08）：当下成败→先看值使；长远发展→先看值符；小事→直接看用神
5. 检查值使门是否受迫/制（R04门宫关系）、是否空亡
6. 符使生克关系：符生使+使生用=必成；符克使+使克用=极难成

#### 第五步：竖看用神落宫（7层分析）

对每个用神落宫，按以下7层**依次分析**。

**分析顺序**（从上到下逐层看）和**判断权重**（决定吉凶冲突时谁说了算）是两回事：

> **分析顺序**：1→2→3→4→5→6→7，逐层看完
> **判断权重**：门 > 格局 > 星 > 神（吉凶冲突时，高权重层覆盖低权重层）
> **三奇定位**：三奇(层7)是加分/减分项，不独立参与权重排序。有三奇=锦上添花/雪中送炭，无三奇不扣分

| 层 | 看什么 | 数据来源 | 分析内容 |
|----|--------|---------|---------|
| **1. 天地盘干** | 天盘干 vs 地盘干 | `盘面[宫].天干` + `盘面[宫].地盘` | 主客关系 + 查81格十干克应(`检测.宫[g].格局`) + **若命中具名格局则标注**（查`geju_xiangyi.json`吉格/九遁/凶格，如"青龙返首""大格"等） |
| **2. 八门** | 门 + 门宫关系 | `盘面[宫].八门` + `检测.宫[g].门迫/宫制` | 门吉凶（查`geju_xiangyi.json`八门象意）+ R04五种关系(迫/制/和/义/比和) + R06(凶门旺相→更凶) |
| **3. 九星** | 星 + 月令旺衰 | `盘面[宫].九星` + `检测.全局.旺衰` | 星吉凶（查`geju_xiangyi.json`九星象意）+ R06(凶星旺相→主流减凶，有分歧) |
| **4. 八神** | 神吉凶 | `盘面[宫].八神` | 查`geju_xiangyi.json`八神象意 |
| **5. 四害** | 空亡/入墓/击刑/门迫 | `检测.宫[g]` | R03优先级: 空亡>入墓>击刑>门迫。击刑/门迫→残余50%；入墓/空亡→残余20% |
| **6. 九宫象意** | 落宫代表什么 | `jiugong_xiangyi.json` → 该宫的"断事要点" | 断"是什么事/涉及什么人/在什么场所"。例：用神落坎一宫→事有险阻/暗中进行/与水相关 |
| **7. 三奇专项** | 是否有乙/丙/丁 | `sanqi_xiangyi.json` | 若落宫天盘有三奇：(a)查象意 (b)升殿检查(乙→震/丙→离/丁→兑=大吉) (c)入墓检查(乙→坤/丙→乾/丁→艮=硬否决) (d)**R19三奇得使**：天盘三奇临地盘值使门所在宫→临吉门大吉/临凶门小吉 (e)玉女守门：值使门临地盘丁奇宫→利婚恋经商 |

**门宫关系口诀**（R04）："吉门被克吉不就，凶门被克凶不起；凶门得生祸难避"

**空亡特殊处理**（R07）：翻先天宫追查，区分旺空(有力量只是暂时不显)vs衰空(真的没有)

**旬首落宫地支关系**（R16）：查R16对照表。力量排序（R17）：刑>冲>害>合

#### 第六步：横看用神间生克（R09落宫五行+R12日干中心+R13首重大局+R14主客排序）

**横看前先检查大局**（R13）：伏吟/反吟状态优先级最高，可覆盖宫间生克判断

核心对比（R09以落宫五行为主，竖看才看天干本身五行）：

1. 日干宫 vs 时干宫 → 求测人 vs 所测事（R12日干为中心）
2. 日干宫 vs 用神宫 → 求测人 vs 事项
3. 相关用神宫之间 → 各方势力对比

| 关系 | 含义 |
|------|------|
| 他生我 | 我得力，吉 |
| 我生他 | 我泄气，付出多 |
| 他克我 | 我受制，凶 |
| 我克他 | 我辛苦但可得 |
| 比和 | 势均力敌，事快 |

**主客判断**（R14层级）：大局(伏吟/反吟) > 门天地盘生克 > 奇仪天地盘生克 > 八神 > 时辰
- 伏吟有反转机制：事已进行中时含义可反转
- 地支关系力量（R17）：刑>冲>害>合

#### 第八步：定应期（R10三步法+R11内外盘）

**R10应期三步法**（编者整合，各部分有据）：

1. **定大范围**：按事项类型（测年→月，测月→日，测日→时辰）
2. **定快慢**（R11）：
   - 内外盘：内盘=近+快，外盘=远+慢（日时干一内一外按"远+慢"）
   - 伏吟=近+慢，反吟=远+快（独立维度，不与内外盘映射）
3. **定数值**：值使门落宫数

**优先级检查**（从高到低）：
1. 空亡 → 冲空/填空之时
2. 马星 → 冲马星之时
3. 墓库 → 冲墓之时
4. 长生旺衰 → 生逢生日，克逢克日
5. 值使门所临地支 → 对应时间

**如果你想跳过 evaluator，你必须：**
1. 明确告诉用户"本次未经独立复核，结论可靠性降低"
2. 在 trace 中记录 `evaluator.result = null`（analyze_traces.py 会统计跳过率并告警）

**复核发现问题 → 定位 → 修正 → 重验：**

| evaluator 发现 | 根因 | 修正方式 |
|----------------|------|---------|
| 用神落宫标错 | 断卦agent看错了盘面JSON | 重新从JSON提取，用宫号而非记忆 |

| 横看生克方向反 | 五行关系搞反 | 重新用落宫五行查 `duanjua_knowledge.json` 五行生克表 |
| 四害遗漏 | 没看完所有用神宫的检测项 | 遍历所有用神宫的 `检测.宫[g]` 全部字段 |
| 吉凶计数与结论矛盾 | 数错了或主观偏向 | 重新列表统计，严格按数量判吉凶 |
| 九宫象意/三奇分析缺失 | 跳过了第五步层6-7 | 补充查表分析 |

修正后不再二次复核（避免死循环），但修正内容需标明"经复核修正"让 Boss 知道改了什么。

**如果 evaluator 和断卦 agent 对某个解读有分歧且双方都有道理**：两种解读都呈现给 Boss，标注各自依据，不替 Boss 做判断。

## 注意事项

- 断卦需要用户提供问题背景，如果信息不足主动追问
- "一物一太极"：一次只聚焦一个问题
- 术数有不确定性，措辞用"盘象显示…""从盘面看…"而非绝对断言
- 断卦是引导思考的工具，最终决策在用户

## 防 Context Anxiety

断卦是长输出任务。**不要因为感觉 context 快满就赶工跳步。** 规则：
- 每步输出完整再进下一步，宁可少分析一个用神也不跳过四害检查
- 如果 context 确实紧张，优先保证：第五步竖看 + 第七步综合 + 结论。可以压缩第零步到第四步
- 绝不跳过 verify 自检和 evaluator 复核
