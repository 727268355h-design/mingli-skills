## 二、伏吟局/反吟局在判断中的权重

### 2.1 古籍论述

**《烟波钓叟歌》**（北宋，奇门遁甲纲领性著作）：

> "**就中伏吟为最凶**，天蓬加着地天蓬"

> "**天蓬若到天英上，须知即是反吟宫**"

汤谓注解：
- 伏吟："不宜用兵，惟宜收敛货财"，"孝服，损人口"
- 反吟："此时不利举兵动众，惟宜散恤仓库之事"。即使遇到吉宿，"**虽得奇门盖之**"也难以化解

来源：[国学一字堂 - 烟波钓叟歌句解](https://www.yzthome.com/thread-9.htm)

### 2.2 权重判断：是"一票否决"还是"重要参考"？

经多源交叉验证，**伏吟反吟的权重介于"一票否决"和"重要参考"之间**，具体取决于伏吟/反吟的范围：

| 伏吟/反吟范围 | 权重级别 | 来源依据 |
|---------------|----------|----------|
| **全局伏吟**（星+门+干均伏吟） | **接近一票否决** | "星门都伏吟，事情从内到外，外到内，**整体都停止不前**"；"测婚姻离婚概率高达百分之90" |
| **星门伏吟** | **重大负面影响** | "星门伏吟，造成事情缓慢拖延，事业进展缓慢达不成理想" |
| **仅星伏吟** | **中等影响** | "表面看事情还在动，但将来容易停止" |
| **仅门伏吟** | **中等影响** | "表面看事情停止，内部还在运作" |
| **仅干伏吟** | **轻度影响** | 天盘干=地盘干，属于局部符号状态 |

**关键例外**：
- "古书对伏吟格大加凶论，其实不能全以为然"——伏吟不是绝对凶 ✅ 多源确认
- "**天显时格**也是伏吟局...一般不为凶，反而为吉"——特殊伏吟格局可转吉 ✅ 多源确认
- "伏吟中遇到蓬+蓬，死+死，辛+辛，癸+癸，为**大凶**"——特定组合加重凶性 ✅ 多源确认
- "若不遇吉门吉格更凶，如遇**吉门吉格为有救**"——吉门可缓解伏吟 ✅ 多源确认

### 2.3 权重定性结论

```
全局伏吟/反吟 ≈ 准一票否决（可被天显时格等极少数特殊格局化解）
星门伏吟/反吟 = 重大负面修正因素（不是否决，但严重影响判断基调）
单层伏吟/反吟 = 中等负面修正因素（需结合其他层综合判断）
```

**反吟与伏吟的区别**：
- 伏吟：主迟、主慢、主停滞、利主不利客
- 反吟：主快、主变、主反复、利客不利主

来源：多篇知乎文章 + 易先生 + 易德轩 + 卢师傅博客

## 三、横看时大局伏吟是否"打折"？

### 3.1 伏吟对横看的影响方式

横看（比较两个宫的生克关系）时，伏吟的影响**不是直接"打折"**，而是通过以下机制生效：

**机制一：定基调——宜静不宜动**

> "伏吟格宜静不宜动，动则招灾惹祸，静则聚集财物。"

这意味着：如果横看的结论是"应该主动出击/积极争取"，而大局是伏吟，则横看结论的可行性大打折扣。用神宫生目标宫（本来利于主动求取），但伏吟局下"**动则招灾**"。

**机制二：定时间——主迟主慢**

> "凡星门伏吟局，在空间上主近，在时间上主慢；凡反吟局，在空间上主远，在时间上主快。"

来源：[知乎 - 奇门遁甲的分析流程](https://zhuanlan.zhihu.com/p/618304212)

即使横看生克关系是有利的，伏吟局下事情也会"拖拖拉拉、拖泥带水"，应期延后。

**机制三：定主客——影响力量对比**

> "星、门伏吟局利主不利客"
> "星、门反吟局利客不利主"

横看时如果涉及主客对比（如日干宫vs时干宫的生克），伏吟局会系统性地偏向"主"方。

### 3.2 具体影响模型（综合推导）

```
横看判断 = 宫间五行生克关系 × 大局修正系数

大局修正系数：
- 无伏吟无反吟：1.0（正常）
- 单层伏吟/反吟：0.8（轻度影响）
- 星门伏吟/反吟：0.5（重大影响）
- 全局伏吟：0.2-0.3（严重影响，接近否决）

额外规则：
- 伏吟局下，"利动"类判断额外打折（×0.5）
- 反吟局下，"利静"类判断额外打折（×0.5）
- 伏吟局 + 天显时格 = 不打折（特殊化解）
```

**注意**：以上系数为综合调研后的推导建议，非古籍原始数值。✅ 方向正确，⚠️ 具体数值需实践校验。

### 4.2 古歌诀来源

多个来源引用了古人编撰的《主客歌》：

> "天盘动用占为客，地盘安静占主穴。细看星宫奇门知，察其刑克吉凶决。分其日月旺相方，更辨其方云气色。假如天蓬加九宫，旺相之月在秋冬。喜逢壬癸亥子日，北方黑气客有功。若还天英加一地，冬时北方主反利。奇门星位仿此推，人在时方分仔细。"

### 4.3 主客判断与伏吟反吟的关系

三种判断主客的维度（多来源一致）：

```
1. 时辰维度：五阳时利客，五阴时利主
2. 格局维度：星门反吟利客，星门伏吟利主
3. 天地盘维度：天盘生地盘利主，地盘生天盘利客；天盘克地盘利客，地盘克天盘利主
```

**三种维度冲突时的处理**：⚠️ 各来源未给出明确优先级规则。实践中可能需要综合三个维度加权判断，但无统一标准。

### 4.4 横看中的主客论应用

来源明确指出：
> "学会了判断奇门遁甲中的利主利客，在分析诸多参数时就可以**分出主次、先后**。判断各种力量的强弱时就有了根据。"

横看两宫生克关系时，主客方向决定了"谁主动"的判断：
- 五阳时 + 反吟：双重利客，客方（主动方）优势最大
- 五阴时 + 伏吟：双重利主，主方（被动方）优势最大
- 五阳时 + 伏吟 / 五阴时 + 反吟：信号矛盾，需看天地盘生克作为第三维度裁决

## 五、优先级体系总结

### 5.1 完整判断层级（综合多来源）

```
第一层：一票否决检查
  ├── 五不遇时？ → 是则全局凶
  ├── 六仪击刑？ → 是则极凶
  └── 三奇入墓？ → 是则百事不宜

第二层：大局基调
  ├── 全局伏吟？ → 宜静不宜动，事主迟慢，利主不利客
  ├── 全局反吟？ → 事主反复快速，利客不利主
  └── 五阳时/五阴时？ → 利客/利主

第三层：竖看（单宫分析）
  ├── 八门吉凶（人和 = 核心参数）
  ├── 格局定性（十干克应）
  ├── 九星吉凶（天时）
  └── 八神吉凶（神助）

第四层：横看（宫间关系）
  ├── 用神宫 vs 目标宫的五行生克
  ├── 主客方向判断
  └── 内盘/外盘定远近快慢

第五层：旺衰终裁
  └── 月令旺相休囚 = 成败的终极裁判
```

### 5.2 "缓从门，急从神，大事看星"的传统层级

来源确认的另一个优先级原则（来自《烟波钓叟歌》体系）：

| 情境 | 优先看什么 | 说明 |
|------|-----------|------|
| 急事 | 八神（值符方位） | 来不及细看，取值符方位行动 |
| 缓事 | 八门 | "诸吉以吉门为主" |
| 大事 | 九星 | "大事看星"，天时能量级别 |

### 5.3 要素排序口诀（来源：知乎 苍炳屹）

> "九宫搭台，八门唱戏；九星配乐，八神助戏；天干字幕，跑不了。"

来源：[知乎 - 奇门遁甲五要素排序](https://zhuanlan.zhihu.com/p/1946868568563099260)

## 七、对引擎实现的建议

### 7.1 横看判断流程建议

```python
def hengkan_judge(source_palace, target_palace, global_context):
    """横看判断流程"""

    # 第一步：检查一票否决
    if global_context.has_veto():  # 五不遇时、六仪击刑、三奇入墓
        return VetoResult()

    # 第二步：确定大局基调
    fuyin_level = global_context.get_fuyin_level()  # 全局/星门/单层/无
    fanyin_level = global_context.get_fanyin_level()

    # 第三步：计算宫间五行生克
    raw_relation = calc_wuxing_relation(source_palace, target_palace)

    # 第四步：应用大局修正
    daju_modifier = get_daju_modifier(fuyin_level, fanyin_level)

    # 第五步：应用主客判断
    zhuke_direction = determine_zhuke(
        time_stem=global_context.time_stem,  # 五阳/五阴
        fuyin_fanyin=fuyin_level or fanyin_level,  # 伏吟利主/反吟利客
        tianpan_dipan=calc_tianpan_dipan_relation()  # 天地盘生克
    )

    # 第六步：旺衰终裁
    wangshuai = get_monthly_wangshuai(source_palace, target_palace)

    return HengkanResult(
        raw_relation=raw_relation,
        daju_modifier=daju_modifier,
        zhuke=zhuke_direction,
        wangshuai=wangshuai
    )
```
